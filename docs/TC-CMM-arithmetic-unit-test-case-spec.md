| 항목 | 내용 |
|------|------|
| **문서 제목** | 사칙연산 단위 테스트 케이스 (예시) |
| **프로젝트명** | 인사관리 앱 시스템 구축 |
| **대상 시스템** | 정산 시스템 |

| **작성자** | 홍길동 |
| **승인자** | 박문수 |

---

## 1. 테스트 개요

| 항목 | 내용 |
|------|------|
| **대표 테스트 ID** | TC-CMM-001 ~ TC-CMM-015 (`CMM`: Common / Core Mathematical Module) |
| **테스트 목적** | 사칙연산(덧셈·뺄셈·곱셈·나눗셈)의 계산 정확도 및 경계·오류 동작 검증 |
| **테스트 기능** | 공통 수학 모듈의 `add`, `subtract`, `multiply`, `divide` (또는 동등 연산) |
| **테스트 환경** | 자바 JDK 버전: 17 / IDE: IntelliJ IDEA 2023.2 / 운영 체제: Windows 10 |
| **테스트 일자** | 2020-09-01 |

---

## 2. 전제 조건

1. 프로그램은 오류 없이 성공적으로 컴파일되어야 합니다.
2. 모든 종속성을 올바르게 설치하고 구성해야 합니다.
3. 테스트 대상 API(또는 정적 메서드)의 시그니처와 반환 타입(`int` / `long` / `BigDecimal` 등)이 명세와 일치해야 합니다.

---

## 3. 성공 / 실패 기준

| 구분 | 기준 |
|------|------|
| **성공** | 모든 테스트 사례가 명세된 예상 결과(또는 예상 예외)와 일치합니다. |
| **실패** | 하나 이상의 테스트 사례가 예상과 다릅니다. |

---

## 4. 특별 절차

- 테스트 결과를 기록하고 이에 따라 본 테스트 케이스 문서를 갱신합니다.
- 실패 건은 즉시 개발팀에 전달하여 조치합니다.

## 5. 단위 테스트 케이스 목록

### 5.1 요약 매트릭스 (입력값 / 예상값)

| 순번 | 표현(개념) | 입력 예 | 예상값 | 비고 |
|:----:|------------|----------|--------|------|
| 1 | `1 + 10` | 1, 10 | 11 | 정상 |
| 2 | `0 + 1` | 0, 1 | 1 | 영 포함 |
| 3 | `0 + 0` | 0, 0 | 0 | 경계 |
| 4 | `0 + (-1)` | 0, -1 | -1 | 음수 |
| 5 | `(-1) + (-10)` | -1, -10 | -11 | 음수+음수 |

*(뺄셈·곱셈·나눗셈은 아래 상세 표의 TC-CMM-006 ~ 015 참고.)*
