## 2. 프로젝트 계약 요약(RED 설계 전제)

| 구분 | 내용 |
|------|------|
| **입력** | `int[][]` 4×4, `0` = 빈칸, `0` 정확히 2개, 값은 `0` 또는 `1~16`, 비0 중복 없음. |
| **출력(성공)** | `int[6]` — `[r1, c1, n1, r2, c2, n2]`, 좌표 **1-index**. |
| **마방진 합** | 4×4 → **34** (10개 라인: 4행+4열+2대각선). |

## 3. Dual-Track RED 구조(목표 상태)

```
TRACK A — UI / Boundary RED          TRACK B — Logic RED
(공개 API·검증·응답 형태)                (순수 도메인: 빈칸·누락·판정·솔브)
         ↓                                      ↓
              둘 다 구현 전에는 실패(RED)
```

- **금지 사항(세션 합의):** 구현 코드 작성, GREEN, REFACTOR. **오직 RED 테스트 설계(및 향후 실패 테스트 코드화 계획)**만 범위에 포함.

## 4. TRACK A — UI / Boundary RED 테스트 설계

각 항목: **Test ID**, **시나리오(Given / When / Then 요약)**, **보호 Invariant·계약**.

| Test ID | 시나리오 | Invariant·계약 보호 |
|---------|-----------|---------------------|
| **UI-RED-01** | **Given** 4×4가 아닌 행렬(크기·행 길이 불일치 등). **When** Boundary 공개 API 호출. **Then** 입력 크기 오류(`E_INPUT_SIZE` 등) 예외, 솔버 미호출. | **I1 / FR-01:** 외곽 4×4만 허용. |
| **UI-RED-02** | **Given** `0` 개수 ≠ 2. **When** 동일 API. **Then** `E_EMPTY_COUNT`. | **I2 / FR-01, BR-15:** 빈칸 개수 전제, 검증 실패 시 도메인 미진입. |
| **UI-RED-03** | **Given** `0` 두 개이나 `1~16`·`0` 밖 값 존재. **When** API. **Then** `E_VALUE_RANGE`. | **I3 / FR-01.** |
| **UI-RED-04** | **Given** 비0 중복. **When** API. **Then** `E_DUPLICATE_NONZERO`. | **I4 / FR-01.** |
| **UI-RED-05** | **Given** 검증 통과 부분 보드(예: PRD TD-OK-01급). **When** 완성 호출. **Then** 성공 시 반환 길이 **정확히 6**. | **FR-05 / BR-11·BR-12:** 성공 스키마 `int[6]`. |
| **UI-RED-06** | **Given** 동일 성공 입력. **When** 완성 호출. **Then** 좌표 `r,c` ∈ [1,4] (1-index). | **FR-02 / FR-05:** 경계 노출 좌표 규칙. |

## 5. TRACK B — Logic RED 테스트 설계

| Test ID | 시나리오 | Invariant·계약 보호 |
|---------|-----------|---------------------|
| **LG-RED-01** | **When** `find_blank_coords(board)` (이름은 구현에 맞게 치환). **Then** 정확히 두 좌표, **row-major** 순서(첫 `0` → 둘째 `0`). | **FR-02 / BR-05:** 빈칸 순서·개수. |
| **LG-RED-02** | **When** `find_not_exist_nums(board)`. **Then** 누락 두 수, **`n_low < n_high`**(오름차순). | **FR-03 / BR-06:** 누락 쌍 순서. |
| **LG-RED-03** | **Given A** 완전 마방진. **Then A** `is_magic_square` → True. **Given B** 한 셀만 깨짐. **Then B** → False. 10라인(4행+4열+2대각)만으로 합 34 판정. | **FR-04 / BR-07·BR-08.** |
| **LG-RED-04a** | **Given** 시도 1만 성공 픽스처. **When** `solution(...)`. **Then** `[r_first,c_first,n_low,r_second,c_second,n_high]`, 1-index, AC-FR-05-01 정합. | **FR-05:** 작은 수→첫 빈칸 우선 시도. |

| **LG-RED-04c** | **Given** TD-NS-01 (양 시도 실패). **Then** `E_DOMAIN_NO_SOLUTION`에 대응하는 실패. | **FR-05 / BR-13.** |
| **LG-RED-04d** | 성공 픽스처에 대해 반환 길이 **6**. | **FR-05** 출력 차원. |
| **LG-RED-04e** | 성공 픽스처에 대해 네 좌표 모두 1~4. | **FR-02** 도메인·경계 인덱스 일치. |

## 7. 후속 권장(코드화 시)

- `pytest` 모듈은 `tests/boundary/` vs `tests/domain/`(또는 PRD 명명 `BT-*` / `DT-*`)로 트랙을 물리적으로 분리한다.
- RED만 유지하려면 **의도적 실패**(미구현 import, `pytest.fail`, 또는 스켈런만 존재)로 커밋 범위를 한정한다.
- GREEN 전에 PRD §12의 테스트 ID와 위 **UI-RED-*** / **LG-RED-*** 매핑 표를 `README.md` §8 또는 별도 트레이서빌리티 표에 동기화한다.
