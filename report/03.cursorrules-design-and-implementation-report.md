# MagicSquare .cursorrules 설계 및 반영 작업 보고서

## 문서 목적
- 본 문서는 MagicSquare 프로젝트의 `.cursorrules` 설계 및 파일 반영 작업 내역을 정리한다.
- 범위는 규칙 초안 작성, 섹션 확장, 실제 파일 생성, 추가 요구사항 반영 시도까지 포함한다.

## 작업 배경
- 목표: Cursor AI가 바로 해석 가능한 프로젝트 규칙 파일 구축
- 대상 파일: `src/.cursor/rules/.cursorrules`
- 기준 아키텍처/방법론: ECB + Dual-Track UI + Logic TDD

## 수행 내역 요약

### 1) 규칙 뼈대 및 섹션 구조 확정
- 최상위 섹션 구조를 확정했다.
  - `project`
  - `code_style`
  - `architecture`
  - `tdd_rules`
  - `testing`
  - `forbidden`
  - `file_structure`
  - `ai_behavior`
- 섹션 간 가독성을 위해 80자 `#` 구분선을 적용했다.

### 2) TDD 규칙 세분화
- `tdd_rules`를 `red_phase`, `green_phase`, `refactor_phase`로 분리했다.
- 각 phase에 대해 `description`, `rules`, `must_not` 형태로 상세화했다.

### 3) 전체 규칙 문서 고도화
- 코드 스타일 정책 추가:
  - Python 3.10+
  - PEP8 엄격 준수
  - 타입힌트 필수
  - Google docstring 필수
  - max line length 88
- 아키텍처 정책 추가:
  - ECB 레이어 책임 정의(boundary/control/entity)
  - 허용/금지 의존성 방향 명시
- 테스트 정책 추가:
  - pytest + AAA 패턴
  - fixture 스코프 원칙
  - 커버리지 기준
- 금지 패턴 정책 추가:
  - `print()` 디버깅 금지
  - 하드코딩 상수 금지
  - `except:` 단독 사용 금지
- AI 동작 정책 추가:
  - 코드 생성 전/중/후 행동 규칙
  - 위반 시 정책

### 4) 실제 파일 생성 반영
- 요청 경로에 규칙 파일을 생성했다.
  - `src/.cursor/rules/.cursorrules`

### 5) 추가 요구사항 반영 진행 상태
- 추가 요청된 도메인 강화 항목(4x4, 입력 검증, 누락값 계산, 합 34, UI/로직 분리, 테스트 우선순위, mock 남용 금지, 명시적 데이터 전달)을 기존 규칙에 보강하는 패치를 준비했다.
- 해당 패치 적용 과정에서 사용자 인터럽트가 발생해 최종 반영 완료까지는 진행되지 않았다.

## 현재 산출물 상태
- 규칙 파일 존재: 예
- 경로: `src/.cursor/rules/.cursorrules`
- 상태: ECB/TDD/테스트/금지 규칙이 포함된 동작 가능한 베이스라인
- 주의: 마지막 추가 요구사항 보강 패치는 인터럽트로 인해 일부/미반영 가능성이 있으므로 재적용 점검이 필요하다.
